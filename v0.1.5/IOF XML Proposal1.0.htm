This proposal for the IOF Standard Information Exchange using Extensible Markup Language(XML) assumes that users of these standards will want some flexibility depending on their situation.  In order to achieve this I have created DataTypeDefinitions (DTD) for single Entities.  A standard set or collection of data such as Results is then created using a complex DTD which simply includes the necessary DTDs for each entity it needs to support.

This proposal extends the work done by the IOF Standards Committee and specifically the proposal to use XML provided by Mr. Ian Watson of GBR.

 

 

<?xml version="1.0" encoding="ISO-8859-1"?>

 

<!ENTITY % Competitor SYSTEM "iofCompetitor.dtd">

%Competitor;

<!ENTITY % CCard SYSTEM "iofCCard.dtd">

%CCard;

<!ENTITY % SplitTimes SYSTEM "iofSplitTimes.dtd">

%SplitTimes;

<!ENTITY % Times SYSTEM "iofTimes.dtd">

%Times;

 

<!ELEMENT Results (Result)*>

<!ELEMENT Result (Competitor,CCard,SplitTimes,Times)>

IofResults.XML contains several entities defined by their individual dtds.

 

<?xml version="1.0" encoding="ISO-8859-1"?>

 

<!ELEMENT Person (Name,BirthDate,Address)>

<!ATTLIST Person Class CDATA #REQUIRED>

<!ATTLIST Person ID CDATA #REQUIRED>

<!ATTLIST Person ClubName CDATA #REQUIRED>

<!ATTLIST Person ClubID CDATA #REQUIRED>

<!ATTLIST Person Sex (M|F) #REQUIRED>

<!ATTLIST Person Nationality CDATA #REQUIRED>

<!ELEMENT Name (Family,Given+)>

<!ELEMENT Family (#PCDATA)>

<!ELEMENT Given (#PCDATA) >

<!ELEMENT BirthDate EMPTY>

<!ATTLIST BirthDate Year CDATA #REQUIRED>

<!ATTLIST BirthDate Month CDATA #REQUIRED>

<!ATTLIST BirthDate Day CDATA #REQUIRED>

<!ELEMENT Address EMPTY>

<!ATTLIST Address Street CDATA #REQUIRED>

<!ATTLIST Address City CDATA #REQUIRED>

<!ATTLIST Address ZipCode CDATA #REQUIRED>

<!ATTLIST Address Country CDATA #REQUIRED>
